📝 1. 組合語言程式的結構

每一行組合語言程式由三個欄位組成：

Label（可省略）｜ Mnemonic ｜ Operand ｜ 註解（.開頭，可省略）
📌 判斷 Label 有沒有的方法 看這行的第一個字元是不是空白（space 或 tab）。
是空白 → 沒有 Label，第一個 token 是 mnemonic
不是空白 → 有 Label，第一個 token 是 label，第二個是 mnemonic

🔧 2. Assembler Directives（虛擬指令）

這些指令不會產生機器碼，是給組譯器看的控制指令：

指令用途範例佔幾 bytes
START程式起始位址PROG START 10000
END程式結束，指定第一條執行指令END FIRST0
BYTE定義 byte 常數EOF BYTE C'EOF'字串長度
WORD定義 word（3 bytes）整數常數THREE WORD 33
RESB保留 N 個 bytesBUFFER RESB 4096N
RESW保留 N 個 words（N×3 bytes）TABLE RESW 100N×3

BYTE 的兩種寫法

EOF    BYTE  C'EOF'     . 字元常數，3 bytes（E, O, F 的 ASCII）
MASK   BYTE  X'F1'     . 十六進位常數，1 byte（= 0xF1）

💻 3. 完整範例：COPY 程式

這是教科書 Fig. 2.1 的完整 SIC 組合語言程式，從輸入裝置讀一筆紀錄，寫到輸出裝置：

主程式呼叫兩個副程式循環讀寫，直到遇到 EOF
. 主程式
COPY   START  1000       . 程式起始位址 0x1000
FIRST  STL   RETADR      . 儲存返回位址
CLOOP  JSUB  RDREC       . 呼叫 RDREC 讀一筆紀錄
       LDA   LENGTH      . 檢查讀取長度
       COMP  ZERO
       JEQ   ENDFIL      . 長度=0 表示 EOF
       JSUB  WRREC       . 呼叫 WRREC 寫出紀錄
       J     CLOOP       . 繼續下一筆
ENDFIL LDA   EOF
       STA   BUFFER
       LDA   THREE
       STA   LENGTH
       JSUB  WRREC
       LDL   RETADR
       RSUB

. 資料定義
EOF    BYTE  C'EOF'
THREE  WORD  3
ZERO   WORD  0
RETADR RESW  1
LENGTH RESW  1
BUFFER RESB  4096

. RDREC 副程式（從輸入裝置讀一筆紀錄到 BUFFER）
RDREC  LDX   ZERO
       LDA   ZERO
RLOOP  TD    INPUT        . 測試裝置是否就緒
       JEQ   RLOOP        . 未就緒，繼續等
       RD    INPUT        . 讀 1 byte 到 A
       COMP  ZERO         . 檢查是否為 null (0x00)
       JEQ   EXIT
       STCH  BUFFER,X     . 存到 BUFFER[X]（索引定址）
       TIX   MAXLEN       . X++，若 X < MAXLEN 繼續
       JLT   RLOOP
EXIT   STX   LENGTH
       RSUB
INPUT  BYTE  X'F1'
MAXLEN WORD  4096

. WRREC 副程式（從 BUFFER 寫一筆紀錄到輸出裝置）
WRREC  LDX   ZERO
WLOOP  TD    OUTPUT
       JEQ   WLOOP
       LDCH  BUFFER,X
       WD    OUTPUT
       TIX   LENGTH
       JLT   WLOOP
       RSUB
OUTPUT BYTE  X'05'
       END   FIRST
💡 TIX 指令說明 TIX m 做兩件事：① X ← (X) + 1，② 比較新的 X 和 (m)，設定 CC
所以 TIX MAXLEN + JLT RLOOP = 迴圈計數器（相當於 C 的 if (++i < maxlen) goto loop）

🔁 4. 常見程式模式

陣列索引迴圈（每次 +3，因為 1 word = 3 bytes）

. C 語言：for(i=0; i<100; i++) gamma[i] = alpha[i] + beta[i];
       LDX   ZERO        . X = 0
LOOP   LDA   ALPHA,X     . A = alpha[i]（索引定址）
       ADD   BETA,X      . A = A + beta[i]
       STA   GAMMA,X     . gamma[i] = A
       TIX   K300        . X += 3，比較 X 和 300（100×3）
       JLT   LOOP
K300   WORD  300
🧠 小測驗
為什麼陣列迴圈每次要把 X 加 3 而不是 1？
查看答案
因為 SIC 的一個 word 是 3 bytes，陣列元素是 WORD 型態，所以相鄰元素之間相差 3 個 byte 的位址。索引 X 要加 3 才能跳到下一個元素。

✏️ 課堂練習 2026 / 03 / 12 & 03 / 19

把高階語言翻成 SIC 組合語言，是本課最核心的能力。

03/12 Q3 · 40%
將以下 C 程式翻譯成 SIC 組合語言（含資料宣告）：
int BETA = 3;
int GAMMA = 8;
int ALPHA;
ALPHA = BETA * GAMMA + 8;
SIC 有 MUL 指令（把 A 乘以 operand，結果存回 A）。
查看解答 
BETA   WORD  3
GAMMA  WORD  8
ALPHA  RESW  1
EIGHT  WORD  8
       LDA   BETA     ; A = 3
       MUL   GAMMA    ; A = 3 × 8 = 24
       ADD   EIGHT    ; A = 24 + 8 = 32
       STA   ALPHA    ; ALPHA = 32
為什麼需要宣告 EIGHT？因為 SIC 沒有立即定址，常數 8 必須先放在記憶體才能用 ADD 讀取。
字元陣列迴圈 · 30%
將以下 C 迴圈翻譯成 SIC 組合語言（空白字元 ASCII = 32）：
char str[10];
for (i = 0; i < 10; i++)
    str[i] = ' ';
char 是 1 byte，用 STCH（store byte）寫入；用 X 暫存器作索引，每次加 1；用 TIX 自動把 X 加 1 並比較。
查看解答 
       LDX   ZERO     ; X = 0（起始索引）
MOV    LDCH  SPACE    ; A 低 8 bits = ' '
       STCH  STR,X    ; str[X] = ' '
       TIX   TEN      ; X = X+1；若 X < 10 設 CC=<
       JLT   MOV      ; 繼續迴圈

STR    RESB  10       ; 10 bytes 的字元陣列
SPACE  BYTE  C' '    ; 空白字元（引號內有一個空格，ASCII = 32）
ZERO   WORD  0
TEN    WORD  10
TIX TEN 做兩件事：① 把 X 加 1，② 比較 X 和 TEN（=10），若 X < 10 就設 condition code 為「less than」。接著 JLT 判斷這個 condition code 決定是否繼續。
整數陣列迴圈 · 30%
將以下 C 迴圈翻譯成 SIC 組合語言：
int arr[10];
for (i = 0; i < 10; i++)
    arr[i] = 0;
int 是 WORD = 3 bytes，索引每次要加 3。TIX 只能加 1，所以改用手動更新索引。終止條件：INDEX ≥ 30（= 10 × 3）。
查看解答 
       LDA   ZERO
       STA   INDEX    ; INDEX = 0
MOV    LDX   INDEX    ; X = 目前索引（byte 偏移量）
       LDA   ZERO
       STA   ARR,X    ; 把 0 寫到 ARR + X 的位址（X 是 byte 偏移量，不是 index）
       LDA   INDEX
       ADD   THREE    ; INDEX += 3
       STA   INDEX
       COMP  THIRTY   ; 比較 INDEX 與 30
       JLT   MOV      ; INDEX < 30 繼續

ARR    RESW  10       ; 10 × 3 = 30 bytes
ZERO   WORD  0
THREE  WORD  3
THIRTY WORD  30
INDEX  RESW  1
和 char 版本的差異：int 每個元素佔 3 bytes，所以終止值是 10×3=30，步長是 3。
