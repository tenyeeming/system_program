⚠️ 容易混淆的地方
PC-relative、Base-relative、Immediate、Indirect、Indexed 都是 Format 3，Loc 一律 +3。
它們的差別只在 nixbpe 旗標和 TA 的算法，不影響指令長度。
唯一會讓 Loc 多加 1 的是 + 前綴（Format 4，+4）。

📍 2. PC-Relative 定址

最常用的 Format 3 定址模式。組譯器計算 disp = 目標位址 − 下一條指令的 PC 值。

disp = 目標位址 − 下一條指令 PC（PC-relative）

範例計算（來自課本）

Loc指令目標位址PC（Fetch後）disp = 目標 − PCnixbpeObject Code
0000STL RETADR00300003002D11001017 20 2D
0017J CLOOP0006001AFEC（負數！）1100103F 2F EC
📌 SIC/XE Format 3 Object Code 組合方式（3 bytes）
Byte 0：opcode[7:2] + n + i（原 opcode 最低兩位改成 ni）
Byte 1：x + b + p + e + disp[11:8]
Byte 2：disp[7:0]

STL opcode = 0x14，ni=11，PC-relative(p=1)，disp = 002D：
Byte0 = 0001 0111 = 17，Byte1 = 0010 | 0000 = 20，Byte2 = 2D → 17 20 2D

J opcode = 0x3C，ni=11，PC-relative(p=1)，disp = FEC（負數）：
Byte0 = 0011 1111 = 3F，Byte1 = 0010 | 1111 = 2F，Byte2 = EC → 3F 2F EC
⚠️ PC-Relative 的限制 disp 是 12-bit 有號數，範圍 -2048 ~ +2047。
如果目標太遠（超出範圍），就要改用 Base-Relative 或 Format 4（+op）。

🏠 3. Base-Relative 定址

用 B 暫存器當基底，disp 為 0 ~ 4095（無號數）的偏移量。

TA = (B) + disp，disp 為 12-bit 無號數（0~4095）

組譯器選擇順序

  1. 1
    先試 PC-relative
    計算 disp = 目標 − PC，若在 -2048~2047 之間 → 使用 PC-relative（p=1）
  2. 2
    若 PC-relative 超出範圍，試 Base-relative
    計算 disp = 目標 − (B)，若在 0~4095 之間 → 使用 Base-relative（b=1）
  3. 3
    若兩者都超出範圍
    必須使用 Format 4（+op m），e=1，有完整 20-bit 位址

⚡ 4. Immediate 與 Indirect 定址

Immediate = 值直接在指令裡；Indirect = 先取位址再取值
🧠 小測驗
計算：指令 160 104E STCH BUFFER,X
BUFFER 位址 = 0x0036，(B) = 0x0033，PC（Fetch後）= 0x1051
請問組譯器會用 PC-relative 還是 Base-relative？disp 是多少？
查看答案
試 PC-relative：disp = 0x0036 − 0x1051 = 負的大數，超出 −2048 範圍 ❌
試 Base-relative：disp = 0x0036 − 0x0033 = 3（在 0~4095 內）✅
→ 使用 Base-relative，b=1，disp = 0x003，nixbpe = 111100（n=1,i=1,x=1,b=1,p=0,e=0）

Object Code（3 bytes）：
STCH opcode = 0x54，ni=11 → Byte0 = 0101 0111 = 57
x=1, b=1, p=0, e=0 → Byte1 高 4 bits = 1100；disp[11:8] = 0x0 → Byte1 = C0
Byte2 = disp[7:0] = 03
→ Object Code = 57 C0 03
