🗺️ 1. 六種定址模式總覽

格式與 Loc 增加量

定址模式決定的是「怎麼算 TA」，但 Loc 增加多少由指令格式決定：

⚠️ 容易混淆的地方
PC-relative、Base-relative、Immediate、Indirect、Indexed 都是 Format 3，Loc 一律 +3。
它們的差別只在 nixbpe 旗標和 TA 的算法，不影響指令長度。
唯一會讓 Loc 多加 1 的是 + 前綴（Format 4，+4）。

📍 2. PC-Relative 定址

最常用的 Format 3 定址模式。組譯器計算 disp = 目標位址 − 下一條指令的 PC 值。

disp = 目標位址 − 下一條指令 PC（PC-relative）

範例計算（來自課本）

Loc指令目標位址PC（Fetch後）disp = 目標 − PCnixbpeObject Code
0000STL RETADR00300003002D11001017 20 2D
0017J CLOOP0006001AFEC（負數！）1100103F 2F EC
📌 SIC/XE Format 3 Object Code 組合方式（3 bytes）
Byte 0：opcode[7:2] + n + i（原 opcode 最低兩位改成 ni）
Byte 1：x + b + p + e + disp[11:8]
Byte 2：disp[7:0]

STL opcode = 0x14，ni=11，PC-relative(p=1)，disp = 002D：
Byte0 = 0001 0111 = 17，Byte1 = 0010 | 0000 = 20，Byte2 = 2D → 17 20 2D

J opcode = 0x3C，ni=11，PC-relative(p=1)，disp = FEC（負數）：
Byte0 = 0011 1111 = 3F，Byte1 = 0010 | 1111 = 2F，Byte2 = EC → 3F 2F EC
⚠️ PC-Relative 的限制 disp 是 12-bit 有號數，範圍 -2048 ~ +2047。
如果目標太遠（超出範圍），就要改用 Base-Relative 或 Format 4（+op）。

🏠 3. Base-Relative 定址

用 B 暫存器當基底，disp 為 0 ~ 4095（無號數）的偏移量。

TA = (B) + disp，disp 為 12-bit 無號數（0~4095）

組譯器選擇順序

  1. 1
    先試 PC-relative
    計算 disp = 目標 − PC，若在 -2048~2047 之間 → 使用 PC-relative（p=1）
  2. 2
    若 PC-relative 超出範圍，試 Base-relative
    計算 disp = 目標 − (B)，若在 0~4095 之間 → 使用 Base-relative（b=1）
  3. 3
    若兩者都超出範圍
    必須使用 Format 4（+op m），e=1，有完整 20-bit 位址

⚡ 4. Immediate 與 Indirect 定址

Immediate = 值直接在指令裡；Indirect = 先取位址再取值
🧠 小測驗
計算：指令 160 104E STCH BUFFER,X
BUFFER 位址 = 0x0036，(B) = 0x0033，PC（Fetch後）= 0x1051
請問組譯器會用 PC-relative 還是 Base-relative？disp 是多少？
查看答案
試 PC-relative：disp = 0x0036 − 0x1051 = 負的大數，超出 −2048 範圍 ❌
試 Base-relative：disp = 0x0036 − 0x0033 = 3（在 0~4095 內）✅
→ 使用 Base-relative，b=1，disp = 0x003，nixbpe = 111100（n=1,i=1,x=1,b=1,p=0,e=0）

Object Code（3 bytes）：
STCH opcode = 0x54，ni=11 → Byte0 = 0101 0111 = 57
x=1, b=1, p=0, e=0 → Byte1 高 4 bits = 1100；disp[11:8] = 0x0 → Byte1 = C0
Byte2 = disp[7:0] = 03
→ Object Code = 57 C0 03

🔢 5. Format 2：暫存器指令（Loc +2）

Format 2 只有 2 bytes = 4 hex 字元，結構：[opcode 8] [r1 4] [r2 4]，完全沒有 nixbpe 或 disp，LOC +2。

暫存器編號（r1 / r2 欄填什麼）

名稱AXLBSTFPCSW
編號（hex） 012345689

Object Code 計算

Loc指令Object Code計算方式
1019ADDR S, X90 41opcode=90，r1=S=4，r2=X=1 → 90 41
101BCOMPR X, TA0 15opcode=A0，r1=X=1，r2=T=5 → A0 15

📌 Format 2 速記
Object Code = [opcode 2 hex][r1 1 hex][r2 1 hex]，總共 4 hex 字元（2 bytes）
r1、r2 直接查暫存器編號表，各佔 1 個 hex digit 拼上去即可

➕ 6. Format 4（+ 前綴）：Object Code = 4 bytes = 8 hex 字元（Loc +4）

指令前加 +（如 +LDB、+STA）→ Format 4，比 Format 3 多 1 byte，LOC +4，Object Code 從 6 hex 變成 8 hex 字元，e=1。

Object Code 計算範例

Loc指令目標位址Object Code（8 hex）計算說明
100C+LDB #TABLE1TABLE1=21C1 69 10 21 C1 LDB opcode=68，immediate(ni=01)→Byte0=69；e=1(xbpe=0001)→Byte1=10；addr=21C1→Byte2=21，Byte3=C1
1020+STA TOTALTOTAL=3937 0F 10 39 37 STA opcode=0C，simple(ni=11)→Byte0=0F；e=1→Byte1=10；addr=3937→Byte2=39，Byte3=37
⚠️ Format 3 vs Format 4 關鍵差異
Format 3：Byte1 高 4 bits = xbpe，低 4 bits = disp[11:8]（disp 共 12 bits）
Format 4：Byte1 高 4 bits = xbpe（e=1），低 4 bits = addr[19:16]（addr 共 20 bits）
絕大多數情況 addr[19:16]=0，所以 Byte1 = 0x10（simple 或 immediate）

⚠️ 7. SIC vs SIC/XE：RSUB Object Code 不同

SICSIC/XE
RSUB Object Code4C 00 004F 00 00
原因舊格式：[opcode 8][x][addr]，opcode=0x4C 直接填入新格式：opcode 低 2 bits 改成 ni=11，0x4C→0x4F
SIC/XE 中所有 Format 3/4 指令的 Byte0 都要重新算：
Byte0 = (original opcode & FC) | ni
RSUB opcode=0x4C=0100 1100，ni=11 → 0100 1111 = 0x4F
