🚀 1. SIC/XE 是什麼？

SIC/XE（eXtra Equipment）是 SIC 的擴充版，增加了更多功能，更接近真實的現代電腦。

比較項目SICSIC/XE
記憶體大小2¹⁵ bytes2²⁰ bytes (1 MB)
暫存器數量5 個9 個
浮點數❌ 無✅ 48-bit
指令格式1 種（24-bit）4 種格式
定址模式Direct, Indexed6 種

📐 3. 四種指令格式

SIC/XE 有 4 種不同長度的指令格式，組譯器根據格式決定指令長度。

Format 3 最常用（nixbpe 旗標）；Format 4 用於超出 12-bit 位移量的情況

nixbpe 旗標說明

旗標名稱說明
nindirect1 = 間接定址（@m）
iimmediate1 = 立即定址（#c）
xindexed1 = 索引定址（m,X）
bbase-relative1 = 以 B 暫存器為基底
pPC-relative1 = 以 PC 為基底
eextended1 = Format 4（20-bit 位址）
⚠️ 重要規則 n=1, i=1 且 x=b=p=e=0 → simple direct addressing（一般直接定址）
SIC 標準模式：n=1, i=1（為了向下相容，SIC 格式翻譯成 SIC/XE 時這樣設）

✏️ 課堂練習 條件判斷翻譯

把 C 語言的 if-else 翻譯成 SIC 與 SIC/XE 組合語言，是最常見的課堂題型。

題目 · 40%
將下列 C 程式片段翻譯成 SIC 組合語言（左欄），再翻譯成 SIC/XE（右欄）：
int num = 49;  char rflag;
if (num == 49)
    rflag = 'T';     // 'T' ASCII = 84
else
    rflag = 'F';     // 'F' ASCII = 70
rflag = 'X';         // 'X' ASCII = 88
SIC 沒有立即定址，所以比較常數需先宣告成 WORD。SIC/XE 可用 # 立即定址或 LDCH #imm。注意：rflag='X' 一定會執行，不管 if 條件是否成立。
查看解答
SIC 版本： LDCH = Load CHaracter，把 1 byte 載入 A 的最低 8 bits；STCH = Store CHaracter，把 A 的最低 8 bits 寫入記憶體 1 byte。char 用這兩個指令，不用 LDA/STA（那兩個是處理 3 bytes 的 word）。 
       LDA   NUM
       COMP  I49       ; 比較 A 與 49
       JEQ   SETTRUE   ; 相等跳去設 'T'
       LDCH  CHARF     ; else：把 'F' 這個字元載進 A
       STCH  RFLAG     ;        再把 A 存到 rflag 的記憶體位置
       J     SETX      ; 跳過 then 區段（否則會掉落執行 SETTRUE）
SETTRUE LDCH CHART     ; then：把 'T' 載進 A
       STCH  RFLAG     ;        存到 rflag
SETX   LDCH  CHARX     ; 一定執行：把 'X' 載進 A
       STCH  RFLAG     ;            存到 rflag

CHARX  BYTE  C'X'      ; ASCII 88
CHART  BYTE  C'T'      ; ASCII 84
CHARF  BYTE  C'F'      ; ASCII 70
NUM    WORD  49
RFLAG  RESB  1
I49    WORD  49
SIC/XE 版本（利用立即定址簡化）： 
       LDA   NUM
       COMP  #49       ; 立即比較，不需宣告 I49
       JEQ   SETTRUE
       LDCH  #70       ; 'F' = 70
       STCH  RFLAG
       J     SETX
SETTRUE LDCH #84       ; 'T' = 84
       STCH  RFLAG
SETX   LDCH  #88       ; 'X' = 88
       STCH  RFLAG

NUM    WORD  49
RFLAG  RESB  1
執行流程（num = 49 時）：COMP I49 → 相等 → JEQ 跳到 SETTRUE → 存 'T' → 自然落下到 SETX → 存 'X'。
執行流程（num ≠ 49 時）：COMP I49 → 不相等 → 不跳，繼續執行 → 存 'F' → J SETX 強制跳過 SETTRUE → 存 'X'。
如果不加 J SETX，num≠49 時會繼續執行 SETTRUE 的程式碼，把 'T' 蓋掉 'F'，這是常見錯誤。
