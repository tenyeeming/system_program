🔢 1. 常見進位系統

電腦底層只有 0 和 1，但我們用不同進位來表示資料。要能在這四種進位之間自由換算。

進位名稱使用的符號範例
2二進位 Binary0, 11010₂ = 10₁₀
8八進位 Octal0–712₈ = 10₁₀
10十進位 Decimal0–910₁₀
16十六進位 Hex0–9, A–FA₁₆ = 10₁₀
💡 為什麼十六進位很重要？ 4 個 bit 剛好可以用一個 hex 字元表示，所以機器碼通常用 hex 寫，例如 opcode LDA = 00。

🔄 2. 進位轉換方法

十進位 → 二進位（除 2 取餘數）

整數部分一直除以 2，記下餘數，由下往上讀就是答案。

除 2 取餘數法（由下往上讀結果）

二進位 ↔ 十六進位（4 bits 一組）

每 4 個 bit 對應 1 個 hex 字元，轉換非常快速。

4 bits ↔ 1 hex 字元（最快速的換算方式）

➖ 3. 負數表示：2's Complement（二補數）

電腦用 2's complement 表示負整數，因為它只有一個零，而且加法電路最簡單。

三種方法比較

方法+3（4-bit）-3（4-bit）問題電腦使用？
Sign-Magnitude00111011+0 和 -0 兩種表示❌
1's Complement00111100+0 和 -0 兩種表示❌
2's Complement00111101無（只有一個零）✅

如何計算 2's Complement

2's Complement 計算步驟：正數 → 取反 → 加 1
🎯 快速技巧 從右邊找到第一個 1，保留它和右邊的所有 0，其餘全部反轉。
例：0001 1000 → 第一個 1 在第 4 位 → 1110 1000
🧠 小測驗
8-bit 2's Complement 中，11111101 是十進位幾？
查看答案
MSB = 1，所以是負數。取 2's complement：00000010 + 1 = 00000011 = 3
所以 11111101 = -3

🌊 4. 浮點數表示法（IEEE 32-bit）

整數不夠用，小數需要浮點數。32-bit 單精度浮點數分成三個欄位：

S = 符號位，E = 偏差指數（真實指數 = E - 127），M = 尾數

範例：17.75₁₀ 轉成 32-bit 浮點數

  1. 1
    轉成二進位
    17 = 10001₂，0.75 = 0.11₂ → 17.75 = 10001.11₂
  2. 2
    正規化（Normalize）
    10001.11 = 1.000111 × 2⁴
  3. 3
    計算各欄位
    S = 0（正數）｜ E = 4 + 127 = 131 = 10000011₂ ｜ M = 00011100000000000000000
  4. 4
    組合結果
    0 10000011 00011100000000000000000

使用 8 位元之 2 補數表示法，(-38)₁₀ = (　　　　　　　　)₂
