One-Pass & Multi-Pass 組譯器

Forward Reference ｜ Load-and-go ｜ EQU

⚡ 1. One-Pass 組譯器

目標：只掃一次原始碼就完成組譯，省去中間檔的 I/O 開銷，速度更快。

主要問題：Forward Reference

⚠️ Forward Reference（向前參考） 指的是使用一個在後面才定義的 Label。
例：第 1 行 JEQ ENDFIL，但 ENDFIL 在第 30 行才定義。
One-pass 到第 1 行時還不知道 ENDFIL 的位址！

兩種 One-Pass 組譯器

類型輸出特點
Load-and-go直接輸出到記憶體執行不寫 Object Program，適合開發測試，組譯後立刻執行
一般 One-pass輸出 Object Program需要特殊處理 Forward Reference（duplicate T records）

🔍 2. 處理 Forward Reference 的方法

One-pass 用 Forward Reference List 回頭填入未定義符號的真實位址
📌 Load-and-go 的 Forward Reference 處理 對於指向指令的 forward reference（如 JEQ ENDFIL），load-and-go 無法消除——因為無法跳到還沒讀到的位址。 但對資料的 forward reference 可以用上述方法處理。

🔢 3. Multi-Pass 組譯器

動機：EQU 指令的問題

EQU 讓你用算式定義符號值：

HALFSZ  EQU  MAXLEN/2       . HALFSZ 的值 = MAXLEN ÷ 2
MAXLEN  EQU  BUFEND-BUFFER  . MAXLEN 的值 = BUFEND - BUFFER
PREVBT  EQU  BUFFER-1       . PREVBT = BUFFER 的前一個位址
...
BUFFER  RESB 4096
BUFEND  EQU  *              . * = 目前位址
⚠️ Two-Pass 的問題 HALFSZ EQU MAXLEN/2 在 Pass 1 掃到這行時，MAXLEN 還沒定義（forward reference in directive）！ Two-pass 無法處理「右邊包含未定義符號的 EQU」。

Multi-Pass 解法

SYMTAB 中每個符號記錄哪些其他符號的值依賴於它（dependency list）。掃多遍，每遍解決能解決的符號，直到全部都定義完。

每個 Pass 解決一層依賴，直到所有符號都有值

※ PROG 和 NUM 都是 2000 嗎？是的。START 設定起始位址後 LOCCTR = 2000，這行本身不佔空間；接著處理 NUM WORD 1000，NUM 得到當前 LOCCTR = 2000，然後 LOCCTR 才推進 3 → 2003。

說明：
• 程式長度 = 3794 - 2000 = 1794₁₆（TABLE RESW 6000 bytes = 1770₁₆ bytes 是關鍵）
• NUM WORD 1000 → 1000₁₀ = 3E8₁₆ → 0003E8
• ADD TABLE,X：TABLE=2018，x=1 → byte1 = 0x80 | 0x20 = 0xA0 → 18A018
• T^003788：ONE(000001) + COUNT(0000C8) 共 6 bytes 連續 ✓
• TOTAL 是 RESW → 無 obj code，T record 在此切斷

1-Pass 的差異（Back-Patching）：
掃到 LDX ZERO 時 ZERO 未知 → 輸出 04 000000，記下位址 2004（byte 1-2 of LDX 指令）
掃到 ZERO 定義（3791）後 → 回去把 2004 位置改成 3791
同理修正 LDA(2007)、ADD TABLE(200A)、TIX COUNT(200D)、STA TOTAL(2013)
