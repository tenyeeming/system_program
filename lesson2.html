🖥️ 1. 什麼是 SIC？

SIC（Simplified Instructional Computer）是教科書設計的假想電腦，用來學習系統程式的核心概念。它夠簡單但包含真實電腦的必要功能。

SIC 有兩個版本：

  • Standard SIC：基本版，功能精簡
  • SIC/XE（eXtra Equipment）：擴充版，下一課介紹
📌 為什麼要學 SIC？ 因為組譯器、載入器等系統程式的設計，都跟機器架構高度相關。學 SIC 後，你能把原理套用到任何真實電腦。

💾 2. 記憶體（Memory）

  • 基本單位：8-bit byte
  • 3 個連續 byte = 1 個 word（24 bits）（注意：SIC 特有！一般電腦是 4 bytes）
  • 記憶體大小：2¹⁵ = 32,768 bytes
  • 位址範圍：0x0000 ~ 0x7FFF（15-bit 位址）
SIC 記憶體：以 byte 為單位定址，3 bytes = 1 word

📦 3. 暫存器（Registers）

SIC 共有 5 個暫存器，每個都是 24 bits。

5 個暫存器，每個 24 bits
⚠️ 常考重點 L 暫存器（Linkage）是 SIC 副程式呼叫的關鍵。JSUB 自動把 PC 存入 L，RSUB 從 L 跳回。 如果副程式呼叫副程式（巢狀），需要先把 L 存到記憶體，否則 L 的值會被蓋掉！
📖 名詞解釋：你可能看不懂的術語
術語全名白話說明
Accumulator
累加器（A）		 — 所有算術運算的「草稿紙」。做 ADD/SUB/MUL/DIV 時，結果自動放進 A，不需另外指定要存哪裡。
TA Target Address
目標位址		 指令真正要存取的記憶體位址。直接定址：TA = address；索引定址：TA = address + (X)。
Linkage
連結（L）		 — 好比書籤。JSUB 呼叫副程式前，把「呼叫完要回去的位址（PC）」夾進 L；副程式做完，RSUB 再從 L 跳回去。
CC Condition Code
條件碼		 存在 SW 裡，只有三種值：<（小於）、=（等於）、>（大於）。COMP 比較 A 和記憶體後設定，讓 JLT/JEQ/JGT 決定要不要跳。
Program Counter
程式計數器（PC）		 — 好比看書時眼睛指向下一個字的游標，告訴 CPU「下一條指令在哪」。每讀完一條指令（3 bytes）就自動 +3。

📐 4. 指令格式（Instruction Format）

SIC 每條指令固定 24 bits（3 bytes），分成三個欄位：

opcode(8) | x(1) | address(15) — 總共 24 bits = 3 bytes
📐 各欄位詳解
欄位位元數位置功能說明
opcode 8 bits bit 23 ~ bit 16 指令類型代碼，告訴 CPU 要做什麼動作。例：0x00=LDA、0x0C=ADD、0x48=JSUB。每種指令各有固定的 opcode，查 OPTAB 即可知道。
x 1 bit bit 15 定址模式旗標（索引位元）。0 → 直接定址，TA = address；1 → 索引定址，TA = address + (X)。在組合語言中寫 STCH BUFFER,X 就是把 x 設成 1。
address 15 bits bit 14 ~ bit 0 操作數的記憶體位址，範圍 0x0000 ~ 0x7FFF，剛好 2¹⁵ = 32768 個位址——這就是為什麼 SIC 記憶體大小是 2¹⁵ bytes。配合 x bit 算出 TA 後，才去記憶體讀寫資料。

兩種定址模式

x bit定址模式目標位址 TA 計算範例
0Direct（直接定址）TA = addressLDA ALPHA
1Indexed（索引定址）TA = address + (X)STCH BUFFER,X

📋 5. 重要指令集

載入 / 儲存

LDA  m   →  A ← (m)           載入 word 到 A
STA  m   →  m ← (A)           把 A 儲存到記憶體
LDX  m   →  X ← (m)           載入到 X（索引暫存器）
STX  m   →  m ← (X)           把 X 儲存到記憶體
LDL  m   →  L ← (m)           載入到 L（連結暫存器）
STL  m   →  m ← (L)           把 L 儲存到記憶體
LDCH m   →  A[低8位] ← (m)[低8位]  載入 1 個字元（byte）
STCH m   →  m[低8位] ← A[低8位]  儲存 1 個字元（byte）

算術運算（結果都存入 A）

ADD  m   →  A ← (A) + (m)
SUB  m   →  A ← (A) - (m)
MUL  m   →  A ← (A) × (m)
DIV  m   →  A ← (A) ÷ (m)

比較與跳躍

COMP m   →  比較 (A) 和 (m)，設定 SW 的 CC（'<', '=', '>'）
JLT  m   →  若 CC = '<' 則跳到 m
JEQ  m   →  若 CC = '=' 則跳到 m
JGT  m   →  若 CC = '>' 則跳到 m
J    m   →  無條件跳到 m

副程式

JSUB m   →  L ← (PC)；PC ← m   呼叫副程式（Jump to SUBroutine）
RSUB     →  PC ← (L)          從副程式返回（Return from SUBroutine）

I/O（每次 1 byte）

TD   m   →  測試裝置 m 是否就緒，設定 CC（'<'=就緒，'='=忙碌）
RD   m   →  A[低8位] ← 從裝置 m 讀入 1 byte
WD   m   →  裝置 m ← A[低8位]  寫出 1 byte
💡 I/O 使用範例（等待裝置就緒再讀取） 
INLOOP  TD   INDEV   . 測試裝置
        JEQ  INLOOP  . CC='=' 表示忙碌，繼續等
        RD   INDEV   . CC='<' 表示就緒，讀取
        STCH DATA    . 儲存讀入的字元
INDEV   BYTE X'F1' . 裝置號碼 F1

⚡ 6. Fetch-Decode-Execute 循環

CPU 執行每條指令都會走過這個循環：

每條指令都走 Fetch → Decode → Execute → Store 四步驟

詳細範例：執行 LDA ONE（ONE 在位址 0x0030，值為 1）

步驟PCIR（指令暫存器）A 暫存器動作
初始狀態0x0033??????—
① Fetch0x003600 00 30???取出指令，PC += 3
② Decode0x003600 00 30???opcode=0x00(LDA), x=0, addr=0x030
③④ Execute+Store0x003600 00 3000 00 01A ← mem[0x0030] = 1
🧠 小測驗
以下 24-bit 指令代表什麼？（提示：opcode 0x00 = LDA，x bit 在第 16 位，(X) = 0x0100）
0000 0000 1000 0011 0000 0000
查看答案
opcode = 00 → LDA
x = 1 → 索引定址
address = 0000 0011 0000 0000 = 0x0300
TA = 0x0300 + (X) = 0x0300 + 0x0100 = 0x0400
→ A ← mem[0x0400]

假設暫存器 X = (2FD)₁₆
① 指令 003030₁₆：target address = (　　　)₁₆，A 載入值 = (　　　　)₁₆
② 指令 00B303₁₆：target address = (　　　)₁₆，A 載入值 = (　　　　)₁₆
SIC 指令格式：byte0 = opcode，bytes 1-2 的最高位 = x-bit，其餘 15 bits = address field。x=1 時 target = address field + X。
查看解答
① 003030₁₆：
opcode = 00（LDA），bytes 1-2 = 0x3030，bit15 = 0 → 直接定址
target = 3030₁₆，A ← memory[3030] = 003600₁₆

② 00B303₁₆：
bytes 1-2 = 0xB303 = 1011 0011 0000 0011₂
最高位（bit15）= 1 → x = 1（索引定址）
把最高位清掉取得 address field：B303₁₆ 的最高位設 0 → 3303₁₆
（技巧：B=1011₂ → 去掉最高位變 011₂ = 3，所以 B303 → 3303）
target = 3303 + 2FD = 3600₁₆，A ← memory[3600] = 103000₁₆
