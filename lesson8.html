程式重定位

Absolute vs Relocatable ｜ Modification Record ｜ Multiprogramming

🖥️ 1. 為什麼需要重定位？

現代電腦通常同時執行多個程式（Multiprogramming），記憶體被分配給不同程式。問題來了：

多程式環境下，程式需要能載入到任意位址（Relocatable）

⚖️ 2. Absolute vs Relocatable 程式

Absolute ProgramRelocatable Program
組譯時所有位址固定（如 START 1000）START 0，相對位址
載入時必須載入到指定位址可載入到任意位址
問題無法共享，浪費記憶體需要 Modification Record 告訴 Loader 哪些要改
Object ProgramH, T, E recordsH, T, M, E records
📌 哪些指令不需要修改？
  • PC-relative：disp 是相對值，載入到哪裡都一樣 ✅
  • Base-relative：disp 是相對值 ✅
  • Immediate addressing：值本身不是位址 ✅
  • Format 4（+op）：含完整 20-bit 位址，載入到不同位址就要修改 ❌
  • SIC 直接定址：含絕對位址，也要修改 ❌

📝 3. Modification Record（M record）

M record 告訴 Loader：「這個欄位裡的位址需要加上載入時的起始位址」。

Loader 收到 M record 後，把該欄位加上程式實際載入的起始位址

哪些指令需要 M record（Loc + Object Code 對照）

以下以可重定位版本（START 0000）為例，說明每條指令是否需要 M record：

組譯 → 連結（合併多個 .obj）→ 載入（決定實際位址，執行 M records）→ 執行

✏️ 課堂練習 M Record 實作

同一支程式分別以「固定起始位址」和「可重定位」兩種方式組譯，比較 Object Program 的差異，釐清 M record 的意義。

程式 TEST · 絕對 vs 可重定位
下列程式分別在 起始位址 3000₁₆（版本 A） 和 起始位址 0000₁₆（可重定位，版本 B） 組譯。
寫出兩個版本的 Object Program，並說明版本 B 為何需要 M records。 
       TEST   START  ????   ← 分別填 3000 和 0000
FIRST  LDX    ZERO
       LDA    ZERO
LOOP   ADD    TABLE,X
       TIX    COUNT
       JLT    LOOP
       STA    TOTAL
       RSUB

M record 格式：M^起始位址^長度（半位元組數）。長度通常是 04（修改 4 個 hex digit = 15-bit address field）。版本 B 中每個有位址欄位的指令都要一個 M record。

版本 B 的 ADD TABLE,X：TABLE=0x0015，x=1 → byte1=0x80|0x00=0x80 → 18 80 15 = 188015（與版本 A 的 18B015 不同，因為起始位址不同）。 M record 的意義：
版本 B 的指令中，每條「有位址欄位的指令」都把 Label 的相對位址（從 0 開始）填進去。
例如 LDX ZERO → object code = 04 0039（ZERO 在 offset 0x0039）。
載入器把程式放到 0x3000 時，必須把所有 M record 指定的欄位都加上 0x3000：
0x0039 + 0x3000 = 0x3039 → 這樣才和版本 A 的結果一樣。

M record 的 04 代表：從該起始位元組起，共 4 個 hex digit（= 15-bit address field）需要修改。
快速辨認 · 哪些指令需要 M record？
對可重定位程式，以下哪些指令需要 M record？
  1. RSUB（operand 為空，object code = 4C0000）
  2. LDA ZERO（ZERO 是程式內定義的 label）
  3. LDA #49（SIC/XE 立即定址，operand 是常數）
  4. WORD 5（資料宣告，值是常數）
  5. STA TOTAL（TOTAL 是程式內定義的 label）
查看答案
① RSUB → 不需要：address field 固定是 0（無 operand），不包含可重定位位址
② LDA ZERO → 需要：address field = ZERO 的相對位址，載入時要加 base
③ LDA #49 → 不需要：立即定址的 operand 是常數值，不是記憶體位址
④ WORD 5 → 不需要：純常數，不是位址
⑤ STA TOTAL → 需要：address field = TOTAL 的相對位址

規則：只有「address field 存放另一個 symbol 的位址」時才需要 M record。常數和 RSUB 的 0000 不算。
